# Fonti Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Fonti, a SwiftUI iOS app that lets designers preview any text rendered in every system font, save favourites, open them full-screen with size/weight controls, and share specimen images.

**Architecture:** Single-target iOS 26+ SwiftUI app. Native Liquid Glass throughout (`.glassEffect()`, glass tab bar, glass buttons). Two-tab `TabView` (Browse + Saved) with an independent `NavigationStack` in each. SwiftData persists saved fonts. Feature-folder layout under `Fonti/`. Logic units (font filtering, display-text rules, trait detection, image rendering) live behind small types tested with Swift Testing; views are verified visually in the simulator after each task.

**Tech Stack:** SwiftUI, SwiftData, Swift Testing (`@Test`), iOS 26 Liquid Glass APIs, `ImageRenderer`, `UIFont`/`UIFontDescriptor`. No third-party dependencies.

**Project facts already verified:**
- `Fonti.xcodeproj` uses `PBXFileSystemSynchronizedRootGroup` — files added under `Fonti/` are automatically part of the target.
- `IPHONEOS_DEPLOYMENT_TARGET = 26.2`, `SWIFT_VERSION = 5.0`.
- No test target yet; Task 1 adds one via Xcode UI.
- Starter files: `Fonti/FontiApp.swift` (keep, modify), `Fonti/ContentView.swift` (delete).

**Source spec:** `docs/superpowers/specs/2026-06-05-fonti-design.md`.

```swift
import UIKit

struct FontFamily: Identifiable, Hashable {
    let id: String          // family name (e.g. "Georgia")
    let displayName: String
}

enum SystemFontProvider {
    static func families() -> [FontFamily] {
        UIFont.familyNames
            .filter { !$0.isEmpty && !$0.hasPrefix(".") }
            .sorted()
            .map { FontFamily(id: $0, displayName: $0) }
    }
}
```

```swift
import Foundation
import Observation

@Observable
@MainActor
final class BrowseModel {
    var input: String = ""
    let fonts: [FontFamily]

    init(fonts: [FontFamily] = SystemFontProvider.families()) {
        self.fonts = fonts
    }

    func displayText(for family: FontFamily) -> String {
        let trimmed = input.trimmingCharacters(in: .whitespacesAndNewlines)
        return trimmed.isEmpty ? family.displayName : input
    }
}
```

The heart button reads the current saved state via `@Query`, and toggles by inserting or deleting a `SavedFont` from the `modelContext`.

- [ ] **Step 1: Update `FontCard` to query saved state and toggle**

Overwrite `Fonti/Features/Browse/FontCard.swift`:

```swift
import SwiftUI
import SwiftData

struct FontCard: View {
    let family: FontFamily
    let displayText: String

    @Environment(\.modelContext) private var modelContext
    @Query private var matches: [SavedFont]

    init(family: FontFamily, displayText: String) {
        self.family = family
        self.displayText = displayText
        let name = family.id
        _matches = Query(
            filter: #Predicate<SavedFont> { $0.familyName == name }
        )
    }

    private var isSaved: Bool { !matches.isEmpty }

    var body: some View {
        VStack(alignment: .leading, spacing: 14) {
            Text(displayText)
                .font(.custom(family.id, size: 28))
                .foregroundStyle(.fontiCream)
                .lineLimit(2)
                .frame(maxWidth: .infinity, alignment: .leading)

            HStack {
                Text(family.displayName.uppercased())
                    .font(.caption2)
                    .tracking(1.2)
                    .foregroundStyle(.fontiCream.opacity(0.65))
                Spacer()
                Button(action: toggleSaved) {
                    Image(systemName: isSaved ? "heart.fill" : "heart")
                        .foregroundStyle(isSaved ? .fontiAmber : .fontiCream.opacity(0.65))
                        .contentTransition(.symbolEffect(.replace))
                }
                .buttonStyle(.glass)
                .accessibilityLabel(isSaved ? "Remove from Saved" : "Save font")
            }
        }
        .padding(18)
        .frame(maxWidth: .infinity, alignment: .leading)
        .glassEffect(in: .rect(cornerRadius: 22))
    }

    private func toggleSaved() {
        withAnimation(.snappy(duration: 0.25)) {
            if let existing = matches.first {
                modelContext.delete(existing)
```

```swift
import UIKit

enum FontTraitSupport {
    static func supportsBold(family: String) -> Bool {
        supports(family: family, trait: .traitBold)
    }

    static func supportsItalic(family: String) -> Bool {
        supports(family: family, trait: .traitItalic)
    }

    private static func supports(family: String, trait: UIFontDescriptor.SymbolicTraits) -> Bool {
        let base = UIFontDescriptor(fontAttributes: [.family: family])
        guard let variant = base.withSymbolicTraits(trait) else { return false }
        let font = UIFont(descriptor: variant, size: 16)
        return font.fontDescriptor.symbolicTraits.contains(trait)
            && font.familyName == family
    }
}
```

The `familyName == family` guard prevents UIKit's substitution behaviour from masking unsupported traits with a different family.

```swift
    private var emptyState: some View {
        Text("Heart a font to keep it here.")
            .font(.body)
            .foregroundStyle(.fontiCream.opacity(0.5))
            .multilineTextAlignment(.center)

    }

    private func delete(_ entry: SavedFont) {

}

#Preview("Empty") {

        .modelContainer(for: SavedFont.self, inMemory: true)
}
```

**Spec deviation (intentional):** the spec asks for swipe-to-delete only. SwiftUI's `.swipeActions` is `List`-only — it does not fire inside a `LazyVGrid` row. Two options:

1. Move Saved to a `List` with custom 2-column row layout — loses the gallery look.
2. Implement a custom `DragGesture`-based swipe — ~40 lines of state per cell.

For v1 we ship a long-press **context menu** with Remove. It costs a single modifier, works with `LazyVGrid`, and is discoverable (cards visibly press in on long-press). If the user wants a real swipe later, replace this task with the custom drag-gesture implementation; the rest of the file (grid, empty state, navigation) is unaffected.
